# Contributing to Proof of Video

Thank you for helping build the most comprehensive side-by-side comparison of AI/LLM-generated 3D worlds! 🎬

## What We're Looking For

- Videos of **3D worlds, scenes, or simulations generated by an AI/LLM pipeline**
- Each submission must clearly state the **exact model name + version** and **software + version** used
- Videos should be uploaded to **YouTube** (unlisted is fine) and embedded via the YouTube ID
- The **prompt or scene description** must be included so others can reproduce it

## How to Submit a Video

### 1. Fork the Repository

Click the **Fork** button on [GitHub](https://github.com/proofof-video/proofof-video) and clone your fork locally.

### 2. Upload Your Video to YouTube

- Title format: `[Tool Name] [Model] — [Short Prompt Summary]`
- Description: include the full prompt, model name+version, software name+version, and date
- Copy the video ID from the URL: `https://www.youtube.com/watch?v=VIDEO_ID`

### 3. Edit `index.html`

#### Option A — Add a video to an existing comparison block

Find the `<div class="comparison-block">` whose prompt matches yours and add a new `<div class="video-card">` inside `<div class="video-grid">`:

```html
<div class="video-card">
  <div class="video-embed">
    <iframe
      src="https://www.youtube.com/embed/YOUR_VIDEO_ID"
      title="Tool Name — Prompt Summary"
      allow="accelerometer; autoplay; clipboard-write; encrypted-media; gyroscope; picture-in-picture"
      allowfullscreen
      loading="lazy"
    ></iframe>
  </div>
  <div class="video-meta">
    <span class="model">Tool / Software Name</span>
    <div>
      <span class="tag model-tag">Model Name + Version</span>
      <span class="tag software">Software Name + Version</span>
    </div>
    <p class="desc">
      Brief description. Link to the tool's repository or website if available.
    </p>
  </div>
</div>
```

#### Option B — Start a new comparison block

Copy the template below and paste it inside `<div class="comparisons">` in `index.html`.
Give the block a unique `id` attribute (e.g. `id="prompt-your-scene-name"`).

```html
<!-- ===== Comparison: Your Scene Name ===== -->
<div class="comparison-block" id="prompt-your-scene-name">
  <div class="comparison-block-header">
    <h3>Your Scene Title</h3>
    <div>
      <span class="prompt-label">Prompt</span>
      <p class="prompt-text">
        "Paste the exact prompt you used here."
      </p>
    </div>
  </div>
  <div class="video-grid">

    <div class="video-card">
      <div class="video-embed">
        <iframe
          src="https://www.youtube.com/embed/YOUR_VIDEO_ID"
          title="Tool Name — Your Scene"
          allow="accelerometer; autoplay; clipboard-write; encrypted-media; gyroscope; picture-in-picture"
          allowfullscreen
          loading="lazy"
        ></iframe>
      </div>
      <div class="video-meta">
        <span class="model">Tool / Software Name</span>
        <div>
          <span class="tag model-tag">Model Name + Version</span>
          <span class="tag software">Software Name + Version</span>
        </div>
        <p class="desc">
          Brief description. Optional link to project.
        </p>
      </div>
    </div>

    <!-- Add more video-card blocks for additional models/tools -->

  </div>
</div>
```

### 4. Open a Pull Request

- Branch name: `add/your-scene-name` or `add/model-name`
- PR title: `Add: [Tool] — [Prompt Summary]`
- In the PR description, briefly describe what model and software you used

## Guidelines

| Rule | Detail |
|------|--------|
| **Reproducible** | Include enough info for others to reproduce your result |
| **Honest labeling** | Use the exact model version (e.g. `gpt-4o-2024-11-20`, not just `GPT-4`) |
| **No marketing** | Submissions should be neutral experiments, not advertisements |
| **No harmful content** | Prompts and videos must not contain offensive, hateful, or NSFW material |
| **YouTube only** | Embed via YouTube for consistent playback and long-term availability |

## Questions?

Open an [issue](https://github.com/proofof-video/proofof-video/issues) — we're happy to help!
